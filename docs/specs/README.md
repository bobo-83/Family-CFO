- M93 safe-to-spend (advisor ignored bills and debt): implemented (2026-07-13, user report from the phone). Asked what was available to spend, the advisor answered liquid cash minus the emergency fund and called the rest "truly available" — ignoring every bill about to fall due and every minimum debt payment owed. Not a model slip: `GROUNDING_RULES` *told* it to "base affordability on liquid assets MINUS emergency_fund_reserved", which names neither bills nor debt and has the model doing its own arithmetic (forbidden everywhere else in this project). New deterministic `calculate_safe_to_spend` (liquid − designated emergency fund − bills due − minimum debt payments, income NOT counted) behind a `get_safe_to_spend` grounded tool; the prompt now forbids deriving spendable money by subtraction. It warns rather than flatters: unmodeled debts say the figure is UNDERSTATED, obligations exceeding cash is a warning not a cheerful number. Migration `0045`. No contract change. **Takes effect only once the box is patched.**

- M94 voice conversations never appeared in the list: fixed (2026-07-13, user report). - M87b voice end-of-utterance fix: implemented (2026-07-13, user report after live-testing the natural voice: "my voice kept being cut off"). M86's flat 1.6 s silence rule sent half-finished questions, because a natural thinking pause is routinely longer than that. `EndOfUtterance` now scales the required pause to how finished the sentence sounds — 1.8 s after terminal punctuation, 3.0 s without it, 6.0 s after a word that promises more ("...and", "...um"), with a hard stop so a trailing-off speaker still gets an answer. Not an echo bug: the engine stops transcribing before the reply is spoken, so the phone never hears itself.
